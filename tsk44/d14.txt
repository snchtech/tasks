Ось **лаконічні та правильні описи**, у стилі службової документації до PL/SQL пакетів (SQL-pkg style), без води, максимально чітко.

---

## ✅ Опис функції **apply_time**

```
/**
 * apply_time
 * 
 * Додає час p_time до дати p_base, залишаючи лише часову компоненту.
 * Використовується для формування точної planned_date_start під час генерації завдань.
 *
 * p_base — базова дата.
 * p_time — значення часу (HH24:MI), з якого береться лише частка доби.
 *
 * return — дата p_base з доданою часовою частиною p_time.
 */
```

---

## ✅ Опис процедури **instantiate_for_day**

```
/**
 * instantiate_for_day
 *
 * Генерує всі завдання із шаблонів, які повинні запускатися у зазначену дату.
 * Використовується у щоденних DB JOB для автоматичної генерації завдань.
 *
 * p_run_date           — дата, для якої виконується генерація (за замовчуванням — SYSDATE).
 * p_service_creator_id — ID користувача, від імені якого створюються завдання (опційно).
 * p_default_status_id  — статус, який можна примусово встановити всім створеним задачам (опційно).
 *
 * Логіка:
 *  - визначає шаблони, що мають спрацювати у p_run_date (ONCE / DAILY / WEEKLY);
 *  - перевіряє відсутність дублюючих завдань (idempotent);
 *  - створює задачі через instantiate_once;
 *  - за потреби оновлює статус і дати.
 */
```
