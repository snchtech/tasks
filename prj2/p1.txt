Коротко: `apex.navigation.dialog.*` працює для **сторінок-діалогів**, а у вас — **Inline Dialog (регіон)**. Тому ловити подію треба не глобальним `apexafterclosedialog`, а подіями **jQuery UI dialog** саме на елементі вашого регіону.

### Надійний спосіб (JS, один раз на сторінці)

У Universal Theme елемент діалогу для Inline Dialog має id `STATIC_ID_dialog`. Якщо ваш регіон має Static ID = `userDialog`, слухати треба `#userDialog_dialog`.

```js
(function initInlineDialogHooks(){
  var dlgId = '#userDialog_dialog'; // для Inline Dialog регіону
  // Якщо з якихось причин елемент інакший — підстрахуємося
  if (!document.querySelector(dlgId)) {
    // іноді збігається з самим static id
    if (document.querySelector('#userDialog')) dlgId = '#userDialog';
  }

  // Єдиний колбек очищення
  function cleanUpAfterClose(){
    // ваш div до очищення:
    const target = document.getElementById('your-div-to-clean'); 
    if (target) target.innerHTML = '';      // або target.textContent = '';
    // зупиняємо таймер, якщо був
    if (window._dlgTimer) { clearTimeout(window._dlgTimer); window._dlgTimer = null; }
  }

  // Після закриття (і по хрестику, і програмно)
  apex.jQuery(dlgId).on('dialogclose', cleanUpAfterClose);

  // Опційно — ще перед закриттям, якщо треба підхопити раніше
  apex.jQuery(dlgId).on('dialogbeforeclose', function(){ cleanUpAfterClose(); });

  // Додатково: коли діалог знищується (рідко, але буває)
  apex.jQuery(dlgId).on('dialogdestroy', cleanUpAfterClose);

  // Робимо колбек видимим, щоб можна було викликати з інших місць
  window._inlineDialogCleanUp = cleanUpAfterClose;
})();
```

### Ваш код з таймером — трохи підправити

Винесіть закриття в функцію, яка також викликає очистку:

```js
function closeUserDialog(){
  apex.theme.closeRegion('userDialog'); // закриє Inline Dialog регіон
  // На випадок, якщо тема/версія не кине 'dialogclose', підхопимо вручну:
  if (typeof window._inlineDialogCleanUp === 'function') {
    window._inlineDialogCleanUp();
  }
}

function toggleDialogSection(isSuccess){
  var ok  = document.getElementById('dlg-success');
  var err = document.getElementById('dlg-error');

  if (isSuccess){
    ok?.removeAttribute('hidden');
    err?.setAttribute('hidden', 'hidden');
  } else {
    err?.removeAttribute('hidden');
    ok?.setAttribute('hidden', 'hidden');

    apex.theme.openRegion('userDialog');

    if (window._dlgTimer) { clearTimeout(window._dlgTimer); }
    window._dlgTimer = setTimeout(closeUserDialog, 7000);
  }
}
```

> Чому `apex.navigation.dialog.close(true)` не допоміг?
> Це API для **сторінок-діалогів** (Dialog Page). У випадку **Inline Dialog** (регіон) треба `apex.theme.openRegion/closeRegion` і слухати `dialogclose` на елементі регіону.

### Альтернатива без коду (декларативно в APEX)

1. Додайте **Dynamic Action** на ваш регіон Inline Dialog.
2. Event: **Dialog Closed** (виберіть сам регіон).
3. True Action: **Execute JavaScript Code** → `document.getElementById('your-div-to-clean')?.innerHTML = '';` + за потреби `clearTimeout(window._dlgTimer);`.

Обидва підходи спрацюють і для автозакриття через 7 секунд, і для кліку по хрестику. Якщо раптом у вашій темі елемент має інший id, в консолі перевірте, що існує `#userDialog_dialog`; якщо ні — підставте реальний селектор діалогу (інколи це сам `#userDialog`).
